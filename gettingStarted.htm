Как начать работать с JSFiddle.

Новый проект (fiddle) или шаблон (boilerplate)

Открытие страницы пустого проекта дает возможность начать проект "с нуля" или создать его из имеющихся шаблонов - "бойлерплейтов":

Рассмотрим панель шаблонов поподробнее:


Слева видим кнопку закрытия панели шаблонов. Правее находится целое поле выбора шаблонов с определенными библиотеками JS (jQuery, Vue, React, React + JSX, Preact, TypeScript, CoffeeScript)   и CSS (SCSS, CSS Grid (responsive), Bootstrap, PostCSS). Надпись над ними "Start with a boilerplate" означает - Начать работу с шаблоном. Ниже поля выбора библиотек находится кнопка "Show boilerplate bar less often?" - "Показывать панель шаблонов пореже?". Клик по ней приведет к закрытию панели шаблонов и для зарегистрированных пользователей, вошедших в систему, она перестанет появляться при каждом включении.

Ввод кода и запуск его в работу
(Entering and running code)

JSFiddle представлен панелями (panel) или вкладками (tab), при переходе на макет вкладок. Имеются четыре панели, в трех из них можно вводить код, а в четвертой будет виден результат работы кода. Вместе они составляют рабочую область проекта.
HTML - код структуры документа, не нужно добавлять теги <body>, <doctype>, <head>, они добавляются автоматически.
CSS - таблица стилей. Можно подключить препроцессор SCSS.
JavaScript - поведение. Имеется множество фреймворков и препроцессоров кода, которые можно использовать.
После ввода кода, просто нажмите кнопку ▷ Run (выполнить) в верхней панели действий и на четвертой панели появится результат.

Сохранение и разветвление кода
(Saving and Forking code)

Save/Update делает то, что и написано, сохраняет новый проект или обновляет существующий (и добавит в его название номер версии).
Fork разделяет существующий проект на новый, начинающийся с версии 0.


Установка проекта в качестве базовой версии
(Setting fiddle as Base Version)

 ⓘ   Это действие доступно только для зарегистрированных пользователей.
При работе с проектом и его обновлении номер его версии будет увеличиваться. Иногда, если код вас удовлетворяет, хорошо бы сохранить его, скрыв номер данной версии, и установить эту версию базовой, т.е. первой по умолчанию.
Например, если имеющийся проект jsfiddle.net/oskar/aba321/23 установить как базовый, то эта 23 версия станет доступной как jsfiddle.net/oskar/aba321.

 ⓘ   При сохранении нового названия (title) или описания (description) проекта, на панели управления не происходит каких-либо изменений, чтобы они появились нужно установить этот проект как базовый (Base).
