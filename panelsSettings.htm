Настройки панелей

Каждая панель имеет свои собственные настройки.

Панель HTML
(HTML panel)

Language – можно выбрать язык документа - HTML или HAML. Значение по умолчанию - HTML.
Doctype – тип документа, т.е. выбор из XHTML 1.0 Strict, XHTML 1.0 Transitional, HTML5, HTML 4.01 Strict, HTML 4.01 Transitional, HTML 4.01 Frameset. Значение по умолчанию - HTML5.
Body tag – обычно используется для изменения стиля, вроде <body class="dark_ui">. Некоторые фреймворки (Dojo) используют его для введения виджетов (widget) и загрузки CSS. По умолчанию - пустая строка.

Панель CSS
(CSS panel)

Language – изменяет препроцессор, выбор из CSS, SCSS, SASS, PostCSS (Stage 0+). Значение по умолчанию - CSS.
Options – дополнительные возможности, например как возможность Normalize CSS. Значение по умолчанию Normalized CSS - нормализованный CSS.

Панель JavaScript
(JavaScript panel)

Language – препроцессор (предварительная обработка), выбор из JavaScript, CoffeeScript, JavaScript 1.7, Babel+JSX, TypeScript, CoffeeScript 2, Vue, React, Preact и т.д. Значение по умолчанию - JavaScript.
Frameworks & Extensions – большое число часто используемых JS фреймворков (и их расширений). Значение по умолчанию - No-libary (pure JS) т.е без применения библиотек (чистый JavaScript, часто этот вариант в шутку называют «ванильным» JavaScript'ом - Vanilla JS).
Примечание переводчика: Фре́ймворк (иногда фреймво́рк; англицизм, неологизм от framework — остов, каркас, структура) — программная платформа, определяющая структуру программной системы; программное обеспечение, облегчающее разработку и объединение разных компонентов большого программного проекта.
Употребляется также слово «каркас», а некоторые авторы используют его в качестве основного, в том числе не базируясь вообще на англоязычном аналоге. Можно также говорить о каркасном подходе как о подходе к построению программ, где любая конфигурация программы строится из двух частей:
  ●   Постоянная часть — каркас, не меняющийся от конфигурации к конфигурации и несущий в себе гнёзда, в которых размещается вторая, переменная часть;
  ●   Сменные модули (или точки расширения).

Load type – способ загрузки JS-кода в редактор - подробности смотрите в следующей таблице:

Тип загрузки
Load type		Описание
Description
On Loadкод упаковывается так, что он будет запущен по событию окна onLoad, т.е. когда загрузилась вся страница, включая стили, картинки и другие ресурсы. Значение по умолчанию.
On DOM Readyкод упаковывается так, что он будет запущен по событию окна onDomReady, т.е. когда браузер полностью загрузил HTML, было построено DOM-дерево, но внешние ресурсы, такие как картинки <img> и стили, могут быть ещё не загружены.
No wrap - bottom of <head>код JavaScript не упаковывается и размещается внутри и внизу тега <head>, т.е. пока не будет выполнен скрипт, содержимое страницы не будет показано.
No wrap - bottom of <body>код JavaScript не упаковывается и размещается внутри и внизу тега <body>, т.е. вначале показывается страница, а затем выполняется скрипт.

Framework <script> attribute – атрибут, добавляемый к тегу <script> включенного фреймворка, т.е. <script src="..." data-angular-custom-thingy>. Значение по умолчанию - ie.data-type="".
