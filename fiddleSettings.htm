Панель настроек проекта

Как уже известно, с левой стороны страницы JSFiddle проекта находится панель настроек проекта.
Здесь находятся следующие настройки:
Метки проекта (Fiddle meta),
Внешние ресурсы (Resources),
Асинхронные запросы (Async requests) (более подробное описание смотрите в отдельной статье Асинхронные запросы) и
Другое (ссылки, лицензия) (Other (links, license)).
Рассмотрим их поподробнее.

Метки проекта
    Fiddle meta

На данный момент метки проекта включают в себя название (title) и описание (description).
Title – добавление названия проекта сделает его доступным на странице вашего публичного профиля. Обычно, когда название не установлено, проект будет наполовину публичным (не отображается в списке общедоступных проектов, но у любого, у кого есть прямая ссылка на проект, есть возможность обращения к нему).
По умолчанию здесь установлено Untitled fiddle - проект без названия.
Description – ну это описание проекта. Оно будет отображаться в списке общедоступных проектов.
По умолчанию здесь установлено No description - т.е. описания нет.
Под метками проекта находится подсказка Add title to make the fiddle visible on your profile page, которая переводится как «Добавьте название проекта, чтобы сделать его видимым на странице вашего профиля».

Внешние ресурсы
    Resources

К проекту можно добавлять внешние ресурсы, такие как CSS и JavaScript.
Если кликнуть по заголовку Resources откроется меню со строкой ввода URL-адреса и кнопкой ⊕, которую можно назвать "Добавить". В поле ввода по умолчанию написано JavaScript/CSS URL , т.е. предлагается ввести URL-адрес JavaScript/CSS ресурса.
Ниже записана подсказка из двух пунктов:
Paste a direct CSS/JS URL - вставьте прямой URL-адрес JavaScript/CSS ресурса. То есть, для добавления ресурса, вводите конкретный адрес ресурса, нажимаете кнопку Добавить ⊕ и ваш проект будет включен конкретный URL-адрес.
Type a library name to fetch from CDNJS - наберите название библиотеки для получения от CDNJS. Здесь просто начните вводить название библиотеки и автозавершение строки предложит библиотеки для вставки, выбираете нужную и нажимаете на кнопку ⊕
 ⓘ   Если хотите использовать динамический ресурс, добавьте фиктивную переменную GET (dummy), то есть вот так http://example.com/dynamically.php?somevar=somevalue&dummy=.css. Это сообщит JSFiddle о признании его в качестве CSS ресурса.


Асинхронные запросы
    Async requests

Функция echo имитирует вызовы асинхронных запросов: /echo simulates Async calls:
(при выборе соответствующего формата, воспользуйтесь следующими подсказками)
JSON: /echo/json/
JSONP: //jsfiddle.net/echo/jsonp/
HTML: /echo/html/
XML: /echo/xml/
Дополнительные сведения смотрите в отдельной статье справки.

Другое (ссылки, лицензия)
    Other (links, license)

Ссылки
    Links

Bug tracker - здесь можно прочитать, а также и написать о замеченных ошибках JSFiddle.
Roadmap (vote for features) - дорожная карта (голосуй за функции). Здесь можно ознакомиться с новыми идеями, узнать какие функции приняты в выполнению, а какие отклонены, что сделано и как идет процесс выполнения. Также здесь можно проголосовать за полезные, на ваш взгляд, функции или против ненужных, по вашему мнению.
About - кратко о JSFiddle и его авторах. Перевод на русский язык этой заметки размещен во «Введении» этой справки.
Docs - ссылка на оригинальную версию (на английском языке) данной справки.
Service status - состояние службы.
